# 技术债务记录

最近更新：2026-06-09

## 当前状态

本文档是 GuguPet 后续技术债务的可发现入口。

本次只记录 UI 系统优化与平台验证过程中发现或确认延期的问题。历史技术债务尚未系统整理，旧计划文档中的债务没有在本次迁移、合并或重新评估。

未整理的历史入口包括但不限于：

- `docs/plans/2026-03-24-technical-debt-resolution.md`
- 其他 `docs/plans/` 下的历史设计和实现计划

如需完整债务治理，应单独安排一次历史债务梳理任务。

## 记录规则

| 字段 | 说明 |
| --- | --- |
| 问题 | 债务或延期事项的简短描述 |
| 影响范围 | 受影响的平台、模块或验证链路 |
| 是否阻塞当前 UI 验证 | 标记该问题是否必须在本轮处理 |
| 建议优先级 | P0 最高，P3 最低 |
| 后续建议 | 推荐的后续处理方式 |

## 本轮必须处理的验证阻塞项

这些问题属于当前 UI 优化任务的一部分，不应作为延期项关闭：

- Web 启动阶段不能因不支持的平台 API 直接崩溃。
- Windows 启动不能因重复建表迁移错误阻塞进入主页面。
- 五个主 Tab 需要能被截图验证。
- 关键数据库迁移路径需要有自动测试覆盖。

## 后续维护建议

- 每次新增延期问题时，优先写入本文件，而不是散落在临时计划中。
- 如果某个债务已有专门设计或实现计划，在表格的后续建议中补充对应文档路径。
- 当债务被解决时，保留记录并标记解决日期，避免重复排查历史问题。
