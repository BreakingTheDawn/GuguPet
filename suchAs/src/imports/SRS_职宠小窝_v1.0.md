## 2. 总体描述

### 2.1 产品视角

《职宠小窝》是一款面向求职期青年及职场新人的治愈型陪伴APP，通过轻拟人电子宠物实现情感陪伴与求职行为记录，并通过"彼岸公园"联机功能延伸用户生命周期。

### 2.2 产品定位

- **核心定位**：轻拟人电子宠物 + 自然语言倾诉 + 求职行为自动记录 + 上岸者低压力社交
- **Slogan**：倾诉即记录，上岸不孤单
- **差异化优势**：无口萌系宠物设计、零操作成本记录、情感陪伴延伸

### 2.3 用户特征

| 用户类型 | 年龄段 | 特征描述 | 核心需求 |
|----------|--------|----------|----------|
| 求职期用户 | 22-35岁 | 裸辞/待业/求职过渡期青年 | 情绪陪伴、求职管理、压力释放 |
| 上岸期用户 | 22-35岁 | 已通过本平台获得Offer的用户 | 身份转型、情感沉淀、社区归属 |

### 2.4 约束条件

#### 2.4.1 技术约束

| 约束项 | 约束内容 |
|--------|----------|
| 前端框架 | Flutter 3.16+ |
| 后端服务 | LeanCloud BaaS平台 |
| 本地存储 | SQLite |
| 数据采集 | GitHub Actions自动化 |
| 最低系统版本 | iOS 12.0+, Android 8.0+ |

#### 2.4.2 合规约束

| 约束项 | 约束内容 |
|--------|----------|
| 数据爬取 | 仅抓取公开岗位信息，不涉及用户隐私 |
| 支付合规 | iOS端必须使用Apple IAP |
| 内容审核 | 联机内容必须通过安全审核 |
| 隐私保护 | 符合《个人信息保护法》要求 |

### 2.5 假设与依赖

| 假设/依赖 | 说明 |
|-----------|------|
| LeanCloud服务稳定 | BaaS平台服务可用性 > 99.9% |
| 第三方招聘平台可访问 | 目标网站结构稳定，反爬机制可控 |
| 用户网络环境 | 用户设备具备网络连接能力 |

### 3.2 F01 宠物倾诉室

#### 3.2.1 F01.01 自然语言输入

**功能描述**：用户通过文字或语音向宠物倾诉求职相关的情绪和行为。

**输入方式**：
| 方式 | 说明 |
|------|------|
| 文字输入 | 键盘输入，无字数限制 |
| 语音输入 | 语音转文字（系统API） |

**交互规范**：
- 输入框位于宠物下方
- 无发送按钮，输入后自动提交（延迟500ms）
- 支持多行输入

**需求ID**：F01.01  
**优先级**：P0  
**验收标准**：
- [ ] 文字输入功能正常
- [ ] 语音输入转文字准确率 > 90%
- [ ] 输入后500ms内触发响应

---

#### 3.2.2 F01.02 意图识别

**功能描述**：系统通过关键词规则引擎识别用户的求职行为和情绪状态。

**可识别求职行为（6类）**：

| 行为类型 | 关键词示例 | 情绪分类 |
|----------|------------|----------|
| 简历投递 | 投了、投递、发简历、海投、投了几份 | 正向 |
| 收到面试 | 面试、邀约、叫我去面试、进面 | 正向 |
| 求职被拒 | 拒了、没通过、不合适、拒信、挂了 | 负向 |
| 拿到Offer | offer、录用、录取、通过了、上班 | 正向 |
| 摆烂/休息 | 没投、躺平、摆烂、没看、不想动 | 中性 |
| 焦虑/低落 | 烦、难过、崩溃、焦虑、迷茫、没用 | 负向 |

**前置否定词过滤**：
- 识别"没投"、"没过"等否定表述
- 区分"投了"与"没投"的语义差异

**需求ID**：F01.02  
**优先级**：P0  
**验收标准**：
- [ ] 6类求职行为识别准确率 > 85%
- [ ] 否定词过滤正确率 > 90%
- [ ] 识别响应时间 < 200ms

---

#### 3.2.3 F01.03 宠物响应

**功能描述**：宠物根据识别结果触发对应动作和气泡回应。

**宠物形象规范**：
- 风格：Q版轻拟人咕咕鸟，2-3头身
- 颜色：浅色系（米白、浅蓝、浅黄）
- 设计：无口设计，仅靠动作+表情表达情绪

**动作与气泡映射表**：

| 用户行为/情绪 | 宠物动作 | 气泡内容 |
|---------------|----------|----------|
| 投递简历 | 翅膀拍拍、点头、递小信封 | 嗯！ |
| 收到面试 | 跳一下、翅膀张开、眼睛变亮 | ！！ |
| 求职被拒 | 低头、耳朵耷拉、蹭一蹭 | 呜… |
| 拿到Offer | 转圈、撒小花/星星、开心扑腾 | ♡♡♡ |
| 摆烂/休息 | 安静坐下、陪蹲、发呆 | … |
| 焦虑/低落 | 轻轻靠过来、摸头动作 | 🫂 |

**需求ID**：F01.03  
**优先级**：P0  
**验收标准**：
- [ ] 6类动作正确触发
- [ ] 动画流畅度 > 30fps
- [ ] 气泡显示位置正确（宠物右侧贴边）

---

#### 3.2.4 F01.04 行为记录

**功能描述**：系统自动将识别到的求职行为静默写入数据库。

**记录内容**：
| 字段 | 说明 |
|------|------|
| 用户ID | 用户唯一标识 |
| 原话内容 | 用户倾诉的原话 |
| 行为类型 | 识别出的求职行为 |
| 情绪类型 | 识别出的情绪 |
| 时间戳 | 对话发生时间 |

**存储策略**：
- 优先存储至本地SQLite
- 核心事件实时同步云端
- 对话记录延迟同步

**需求ID**：F01.04  
**优先级**：P0  
**验收标准**：
- [ ] 记录成功率 100%
- [ ] 本地存储响应 < 50ms
- [ ] 云端同步成功率 > 99%

---

#### 3.2.5 F01.05 宠物记忆（Pro版专属）

**功能描述**：宠物记住最近20轮对话中的求职相关关键信息。

**记忆内容**：
- 面试时间/公司
- 求职方向
- 近期情绪状态
- 用户提及的小目标

**触发逻辑**：
- 记忆不影响气泡内容
- 仅影响宠物动作倾向（如用户说明天有面试，后续倾诉时宠物表现出"担心、鼓励"的动作）

**需求ID**：F01.05  
**优先级**：P1  
**验收标准**：
- [ ] 记忆容量 = 20轮
- [ ] 关键信息提取准确率 > 80%
- [ ] 记忆匹配响应时间 < 100ms
